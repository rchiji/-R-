1. データ成型

RNAseqに限らず解析ツールごとに入力データ形式が決まっており、自身のデータをそれらに近づけていくスキルが求められます。
遺伝子発現解析では行に遺伝子、列にサンプルという行列データを扱います。ここでは強大なデータフレームの処理を紹介します。


1-1. デモデータの読み込み

RNAseqの発現量算出まで行った処理ファイルをデモデータとして使います。まずはこちらを読み込みます。

sample1 <- read.delim(file = "RNAseq_data/A1.genes.results", header = T, sep = "\t")
sample2 <- read.delim(file = "RNAseq_data/A2.genes.results", header = T, sep = "\t")
sample3 <- read.delim(file = "RNAseq_data/A3.genes.results", header = T, sep = "\t")

6万行 × 7列のデータです。

dim(sample1)
## [1] 60649     7

先頭を見てみます。

head(sample1)

遺伝子のIDや名前の情報、遺伝子の長さの情報、遺伝子発現量の情報が含まれています。
必要なのは遺伝子の識別列と発現量の列です。

gene_id列とTMP列のみにします。TPMはサンプル間でシークエンスデータ量と遺伝子長の補正を行った後のデータです。

sample1 <- sample1[,c("gene_id", "TPM")]
sample2 <- sample2[,c("gene_id", "TPM")]
sample3 <- sample3[,c("gene_id", "TPM")]


1-2. データフレームの結合1

サンプルごとの発現データをまとめて、遺伝子×サンプルの行列にしていきます。
単純に列方向にサンプルの発現量をつなげていけば良さそうですが、遺伝子の並びは同じなのでしょうか。
6万行もある遺伝子の順番が全サンプルで同じかどうか目視で確認することは行いません。

identical機能を使えば、要素の中身と順番が全く同じなのかを判定できます。

identical(sample1$gene_id, sample2$gene_id)
## [1] TRUE
identical(sample1$gene_id, sample3$gene_id)
## [1] TRUE

各サンプルのgene_id列は全く同じ情報であることが分かりました。

cbind機能でデータフレームやベクトルを列方向に連結することができます。

mat <- cbind(sample1, sample2, sample3)
head(mat)

gene_id列は行名に移動させて、後は削除します。

rownames(mat) <- mat$gene_id
mat <- mat[, c(2,4,6)]
colnames(mat) <- c("sample1", "sample2", "sample3")
head(mat)

これで発現マトリクスが完成しました。

もしくはdata.frame機能で新たに作り直しても良いです。

# 列名を指定しながらdata.frameを作成
mat <- data.frame(sample1 = sample1$TPM,
                  sample2 = sample2$TPM,
                  sample3 = sample3$TPM,
                  row.names = sample1$gene_id # 行名指定
                  )
head(mat)

cbindの際にTPM列だけ繋げても良いです。

# 列名を指定しながらTPM列を横につなげる。行名指定引数は無い。
mat <- cbind(sample1 = sample1$TPM,
             sample2 = sample2$TPM,
             sample3 = sample3$TPM
             )
rownames(mat) <- sample1$gene_id
head(mat)

1-3. データフレームの結合2

遺伝子順が揃っていない場合のデータフレームの結合方法を紹介します。

matchを使った方法

# デモ用に遺伝子順をシャッフルしておきます。
set.seed(0)
sample3 <- sample3[sample(x = seq(nrow(sample3))),]
identical(sample1$gene_id, sample3$gene_id)
## [1] FALSE
head(sample3)

match機能では2つのベクトルを対応箇所を教えてくれます。

head(match(sample1$gene_id, sample3$gene_id))
## [1]  9215 27200 25544 29118 22481 47616

sample1の1行目はsample3の21950行目にあることがわかります。

sample1[1,]
sample3[21950,]

この情報を使ってsample3の行順を入れ替えてやると元通りに戻ります。

gene_order <- match(sample1$gene_id, sample3$gene_id)
sample3 <- sample3[gene_order,]
head(sample3)
identical(sample1$gene_id, sample3$gene_id)
## [1] TRUE

並びが揃った後はcbindなどで単純に結合すればよいです。

dplyrパッケージのjoinシリーズ

2つのデータフレームのある列をkeyにして、並び替えてdata.frameを結合するところまで行ってくれます。

left_join, inner_join, full_join などがありますが、それぞれマッチしなかった行の処理が変わります。

  • left_joinは1つの引数のデータフレームを基準に2つの引数のデータフレームを横つなぎにしますが、1つ目にはあって、2つ目にはない行にNAが入ります。
  • inner_joinは2つのデータフレームに共通する行のみを残します。どちらかに欠損があるとその行が消されてしまいます。
  • full_joinは全ての行を残して、欠損にNAを残します。
library(dplyr)
## Warning: パッケージ 'dplyr' はバージョン 4.2.2 の R の下で造られました
## 
##  次のパッケージを付け加えます: 'dplyr'
##  以下のオブジェクトは 'package:stats' からマスクされています:
## 
##     filter, lag
##  以下のオブジェクトは 'package:base' からマスクされています:
## 
##     intersect, setdiff, setequal, union
# デモ用に遺伝子順をシャッフルしておきます。
set.seed(1)
sample2 <- sample2[sample(x = seq(nrow(sample2))),]

x=, y=にそれぞれのデータフレームを指定し、by=にkeyとなる列のなめを指定します。

mat <- left_join(x = sample1, y = sample2, by ="gene_id")
head(mat)

joinは2つのデータフレームしか対応していないため、3つ以上の場合はmatオブジェクトを更新するように繋げていくと良いです。

mat <- left_join(x = mat, y = sample3, by ="gene_id")
head(mat)
rownames(mat) <- mat$gene_id
mat$gene_id <- NULL
colnames(mat) <- c("sample1", "sample2", "sample3")
head(mat)


1-4. 遺伝子ID/遺伝子名の扱い

この例の行名は遺伝子ID_遺伝子名となっていますが、無駄に長いので遺伝子名だけに修正します。

この例のENSGから始まる遺伝子IDはEnsemblというデータベースで管理されている遺伝子IDです。その他にもENTREZ IDやRefseq IDなどがあります。

アンダーバーで遺伝子IDと遺伝子名が繋がっているので、アンダーバーで要素を区切れば遺伝子名が取得できます。
文字区切りはstrsplit機能がデフォルトで実装されていますが、少し使いづらいのでstringrパッケージのstr_split機能を使用します。

symbol <- 
  stringr::str_split(string = rownames(mat), 
                     pattern = "_", 
                     simplify = TRUE # 戻り値がdata.frameになるオプション
                     )
head(symbol)

rowSums機能で行の合計を集計します。(rowMeansやcolSums, colMeans機能もあります。)

head(rowSums(mat))

行の合計を降順に並び替えた遺伝子順を採用します。 orderで発現値を基に並び替えます。デフォルトでは昇順なのでdecreasing = TRUEにして降順に並び替えます。

geneorder <- order(rowSums(mat), decreasing = TRUE)
mat <- mat[geneorder,]
head(mat)

この並びの場合のduplicateを調べ直して、重複が無い箇所のみを取得します。

symbol <- stringr::str_split(string = rownames(mat), pattern = "_", n = 2,simplify = TRUE)[,2]

# duplicateの出力を!で否定
symbol_unique <- !duplicated(symbol)

# TRUEの箇所のみ抽出
mat <- mat[symbol_unique,]
symbol <- symbol[symbol_unique]

rownames(mat) <- symbol

head(mat)
dim(mat)
## [1] 59429     3

やっと発現マトリクスが完成しました。



2. RNAseqデータのプロファイリング

発現マトリクスからサンプル間のプロファイルの違いを示すことができます。

サンプル数を増やしたデモデータを作成します。

# ファイルパス一覧を取得
filepaths <- dir(path = "RNAseq_data/", pattern = "*results", full.names = T)

# ファイルの読み込みと処理の繰り返しを実行。出力はリスト
dat <- lapply(filepaths, function(x){
  x <- read.delim(file = x, header = T, sep = "\t")
  x <- x[,c("gene_id", "TPM")]
  rownames(x) <- x[,1]
  x[,1] <- NULL
  return(x)
})
# リストの要素をcbind
mat <- do.call(what = cbind, dat)
# 列名を付け直し
colnames(mat) <- paste0("sample", seq(6))

# 発現量で降順にソート
mat <- mat[order(rowSums(mat), decreasing = TRUE),]

symbol <- stringr::str_split(string = rownames(mat), pattern = "_", n = 2,simplify = TRUE)[,2]
# duplicateの出力を!で否定
symbol_unique <- !duplicated(symbol)

# TRUEの箇所のみ抽出
mat <- mat[symbol_unique,]
symbol <- symbol[symbol_unique]

# 行名に遺伝子名を入れる
rownames(mat) <- symbol
head(mat)

2-2. 低発現遺伝子のフィルタリング

分位数やヒストグラムで遺伝子発現量の分布を確認してみると、殆どの遺伝子が低発現な一方で、発現量が高い遺伝子は10000を超えた発現量を持っています。

summary(rowMeans(mat))
##      Min.   1st Qu.    Median      Mean   3rd Qu.      Max. 
##     0.000     0.000     0.000    16.825     0.857 12008.885
hist(rowMeans(mat), breaks = 100)


発現量が全くない = 0のものを調べてみます。

table(rowMeans(mat) == 0)
## 
## FALSE  TRUE 
## 24375 35054

半分以上の遺伝子は発現量が0であることがわかりました。


これらは意味を持ちませんので、解析から除きます。

mat <- mat[rowMeans(mat) != 0, ]
dim(mat)
## [1] 24375     6


さらに発現のスケールを正規分布に近づけるために対数変換しておきます。値が0の箇所は対数変換できませんので、pseudocountとして一律に1を足しておきます。

mat_log <- log2(mat + 1)
summary(rowMeans(mat_log))
##      Min.   1st Qu.    Median      Mean   3rd Qu.      Max. 
##  0.002393  0.232999  1.641814  2.505106  4.494453 13.519271
hist(rowMeans(mat_log), breaks = 100)

発現分布の偏りが減少してきました。 しかし、まだまだ低発現遺伝子が最も多い状態には変わりません。

一般に、RNAseqではこれら低発現遺伝子もフィルタリングして解析に臨みます。もしかすると、低発現領域の遺伝子に大事な物があるのかもしれませんが、低発現遺伝子はphenotypeに影響は与えていないという想定の下で除くのは妥当な処理です。

カットオフ値は任意です。減らし過ぎず、残し過ぎない値を採用します。

table(rowMeans(mat_log) > 1)
## 
## FALSE  TRUE 
## 10487 13888
table(rowMeans(mat_log) > 2)
## 
## FALSE  TRUE 
## 12901 11474
table(rowMeans(mat_log) > 5)
## 
## FALSE  TRUE 
## 19685  4690

ここではカットオフを1に設定してみます。

mat_log <- mat_log[rowMeans(mat_log) > 1,]
dim(mat_log)
## [1] 13888     6
summary(rowMeans(mat_log))
##    Min. 1st Qu.  Median    Mean 3rd Qu.    Max. 
##   1.000   2.601   4.194   4.189   5.484  13.519
hist(rowMeans(mat_log), breaks = 100)

pheatmap(mat = mat_log,
         cluster_rows = F,
         cluster_cols = F,
         show_rownames = F)

何となくsample1, sample2, sample4が近く、sample3, sample5, sample6と は違ったプロファイルであることが見えてきました。


2-3. SDを用いた遺伝子のフィルタリング

サンプル間の違いを示すには上記までの遺伝子数はまだまだ多いと考えられます。 サンプルによって遺伝子発現量が違うのであれば、その遺伝子の分散や標準偏差は高くなるはずです。一方、内因性コントロルのような遺伝子は高発現であってもサンプル間での差は小さいため、サンプルの違いを示すには必要無いと考えられます。


遺伝子ごとの標準偏差を計算します。rowSD機能は無いので1行ごとにsdを繰り返します。 行列の繰り返しにはapplyを使用します。

MARGIN = 1で行ごと、MARGIN = 2で列ごとに処理を繰り返します。繰り返す処理内容はFUN=で指定します。

# 行ごとにsd()を実施
gene_sd <- apply(X = mat_log, MARGIN = 1, FUN = sd)
head(gene_sd)
##    EEF1A1   MT-ND4L   MT-ATP8    MT-ND3    MT-CO2     RPS27 
## 0.3246867 0.6496367 0.3450002 0.3604278 0.6021013 0.3680607
summary(gene_sd)
##    Min. 1st Qu.  Median    Mean 3rd Qu.    Max. 
## 0.03429 0.41230 0.61704 0.75320 0.93545 5.01346
hist(gene_sd, breaks = 100)

殆どの遺伝子が標準偏差1以内であり、サンプル間で発現が変動していないことが伺えます。

2-4. 主成分分析

遺伝子数(1万4千) × サンプル という多次元でサンプル間の違いを可視化するのは困難です。

主成分分析では多次元の遺伝子情報のうち、サンプル間の違いを説明するうえで重要な成分を抽出します。重要な成分ほど上位の成分となります。基本は第一成分と第二成分を用いてサンプル間の違いを示します。

Rで主成分分析をするにはprcomp機能を使用します。※ 列に配置した情報から成分を分析します。遺伝子の情報を成分分析したいのでt機能で行列を転置して、サンプル × 遺伝子 の行列にしてから使用します。

pca_result <- prcomp(x = t(mat_log))
str(pca_result)

2-5. tSNE, UMAP

最近、流行りのtSNEやUMAPもRで簡単に実行することができます。主成分分析と異なり、多次元を2次元に落とし込むためサンプル間の違いを示す際に成分軸を選ぶ必要がありません。

library(Rtsne) # install.packages("ggrepel")

# tSNEは乱数を使うのでseed値を固定します
set.seed(0)

# tSNEの実行
tsne_result <- Rtsne(X = t(mat_log),
                     perplexity = 1 # サンプル数が少ない時は下げる
                     )

ggplotで結果を確認します。

df <- data.frame(tsne_result$Y, row.names = colnames(mat_log))
ggplot(data = df, 
       aes(x = X1, y = X2, label = rownames(df))) + 
  geom_point() + 
  geom_text_repel() # ラベルの重なりを防いでラベル表示

library(uwot) # install.packages("uwot")

どの次元削減方法も一長一短で、どれがベストとかはありません。
